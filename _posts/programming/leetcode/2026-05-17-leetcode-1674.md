## Introduction
* Link: [https://leetcode.com/problems/minimum-moves-to-make-array-complementary](https://leetcode.com/problems/minimum-moves-to-make-array-complementary)

The problem requires to return the minimum number of replacement as an `int` value to make the array `nums` complementary.

> DISCLAIMER: Below solution spoiler is available.
{: .prompt-warning }

## ❌ Idea 1
Thinking the greedy algorithm, the most frequent sum value of `nums[i] + nums[n-1-i]` would lead to the minimum replacements.

The idea had failed to handle all edge cases well, so it was discarded later.

### Trial

The Code A and Code B fail some cases where the most frequent sums do not make the minimum number of replacements. I tried to handle some more cases, where all of the most frequent sums make more replacements than the value which does not derive from any pairs from `nums`. However it still fails some other edge cases.

```cpp
class Solution {
public:
    int minMoves(vector<int>& nums, int limit) {
        const int n = nums.size();
        vector<int> vec_sum(limit * 2 + 1, 0);

        // Find the sum with the most counts
        for(int i=0; i<n/2; ++i){
            ++vec_sum[nums[i] + nums[n-1-i]];
        }

        stack<int> max_nums;
        int max_count = 0;

        for(int i=0; i<vec_sum.size(); ++i){
            int s = vec_sum[i];
            if(s > max_count){
                max_nums = stack<int>();
                max_nums.push(i);
                max_count = s;
            }
            else if(s == max_count){
                max_nums.push(i);
            }
        }


        int answer = numeric_limits<int>::max();
        // Compare the pairs to count the replacements
        while(!max_nums.empty()){
            int k = max_nums.top();
            max_nums.pop();
            int rep_count = 0;

            for(int i=0; i<n/2; ++i){
                int s = nums[i] + nums[n-1-i];
                
                if(k != s){

                    int flag = 0;

                    int a = k - nums[i];
                    int b = k - nums[n-1-i];

                    if(a >= 1 && a <= limit){
                        flag |= 1;
                    }
                    if(b >= 1 && b <= limit){
                        flag |= 1;
                    }

                    rep_count += (2 - flag);
                }
            }

            if(answer > rep_count)
                answer = rep_count;
        }
```
```cpp
        return answer;
    }
};
```

## ❌ Idea 2
I used brute-force here to avoid any complicated edge cases. Instead of just using the most frequent sums as inputs, I used all available sums from the `nums`. I also compared the sums to all `nums` pairs to determine how many replacements should be needed for a pair.

### Trial

## ✅ Idea 3
I thought the Idea 2 would not be optimized well to pass the time limit. After putting some landmark values from a pair on a number line, I finally got the problem solution.

![ranges](ranges.png)

### Trial
I used indirect sorting to put sets of $a+1$, $a+b$, and $b+limit$ from all number pairs into a $3×(2·limit)$ matrix, `graph`. In the matrix, the first row has the counts of $a+1$ values, the second row has the counts of $a+b$ values, and the last row has the counts of $b+limit$ values. In a nutshell, every column is explained like these:

* `graph[x][0]`: The common sum range that needs 1 replacement for the `x`-th pair starts from `graph[x][0]`, which is `a+1`.
* `graph[x][1]`: As it is the same as `a+b`, the pair does not need to be replaced at all here. However any other common sums before and after `graph[x][1]` will need some additional replacements from this pair.
* `graph[x][2]`: The common sum range that needs 1 replacement for the `x`-th pair ends at `graph[x][2]`.

And any other common sum values need 2 full replacements, where both `a` and `b` should be changed to make them.

I defined the minimum number of replacement to make all pairs to `reps`, where `i` is the current common sum to make with all number pairs. We can think this situation as graphing `i` on a number line.

![idea3](idea3.png)

As initializing `reps = n` and iterating `i`, we can track `reps` value by following these steps:

* If `graph[x][0] = i`, decrease `reps` by 1 at `i`.
* If `graph[x][1] = i`, decrease `reps` by 1 at `i`. And then increase `reps` at `i+1`.
* If `graph[x][2] = i`, increase `reps` by 1 at `i+1`.

Among all iterations, the minimum value of `reps` becomes the answer, which is the minimum number of replacements for making a common sum.

#### Code D
> **Accepted**
>
> Runtime: 211 ms (Beats 6.70%)
>
> Memory: 308.37 MB (Beats 5.09%)
{: .prompt-tip}

```cpp
class Solution {
public:
    int minMoves(vector<int>& nums, int limit) {
        const int max_limit = pow(10, 5);
        vector<vector<int>> graph(3, vector<int>(2 * max_limit + 1, 0));
        
        const int n = nums.size();
        const int hn = n / 2;

        for(int i=0; i<hn; ++i){
            int a = (nums[i] < nums[n-1-i] ? nums[i] : nums[n-1-i]);
            int b = (nums[i] < nums[n-1-i] ? nums[n-1-i] : nums[i]);

            ++graph[0][a+1];
            ++graph[1][a+b];
            ++graph[2][b+limit];
        }

        int answer = numeric_limits<int>::max();
        int reps = n;

        for(int i=0; i<graph[0].size(); ++i){
            reps -= graph[0][i];
            if(answer > reps - graph[1][i]){
                answer = reps - graph[1][i];
            }
            reps += graph[2][i];                
        }

        return answer;
    }
};
```

## ✨ Optimization
### Revision 1
Instead of creating a 2D matrix `graph` to store information of every pair, **create an 1D array `x_axis` that works like a number line**. It has the size $2 × (10^5) + 2 = 200002$ and it is initialized to $0$. I set its size to $200002$ because the maximum sum in the problem is $2 × 10^5$, and I want to allow the indices up to $2 × 10^5 + 1 = 200001$ for the largest `b+limit+1` value.

```cpp
int x_axis[200002] = {0};
```

Additionally, I picked up C-style `array` instead of `vector` to reduce overheads.

### Revision 2
I **defined the array `x_axis` as a global variable**. Global variables are allocated in the BSS/Data segment of the program's memory, not on the stack (small and can overflow) or heap (which requires slow system calls via `new` or `malloc`).

### Revision 3
Some programming tests such as LeetCode and SWEA run the solution process or class for the multiple test cases. Therefore we should reset the global variable `x_axis` every time before running other main statements. **Using `std::fill` is easier to efficiently reset it** than writing a manual for-loop.

```cpp
fill(x_axis, x_axis + max_x + 2, 0);
```

### Revision 4
**Make the better code for a compiler to guarantee the code optimization.** Instead of repeating to get the same values from the array `nums`, either assign them to variables `v1` and `v2`, or simply use `std::minmax`. While all perform the same when optimized well, sometimes the compiler becomes afraid of optimizing the former when it is not sure on not ruining any data.

```cpp
for(int i=0; i<hn; ++i){
    int v1 = nums[i];
    int v2 = nums[n-1-i];

    int a = v1 < v2 ? v1 : v2;
    int b = v1 < v2 ? v2 : v1;
}
```

### Revision 5
**Store increments or decrements at a point `x` to `x_axis[x]`.** I also changed to increase the replacement count by 1 at `a+b+1` and at `b+limit+1`; instead of decreasing any `a+b` counts, check if the current `reps` is the minimum value, and then revert the `reps` decrement for operations around `a+b`.

```cpp
--x_axis[a+1];
--x_axis[a+b];
++x_axis[a+b+1];
++x_axis[b+limit+1];
```

### Revision 6
As I changed the main algorithm on the `reps` increment/decrement from the Revision 5, I can simply track the `reps` changes by reading the array `x_axis`, from the minimum possible sum `x = 2` to the maximum possible sum `x = max_x = 2 * limit`.

```cpp
for(int x = 2; x <= max_x; ++x){
    reps += x_axis[x];
    if(reps < answer)
        answer = reps;
}
```

```cpp
static const int optimizer = [](){
    ios::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);

    return 0;
}();

int x_axis[200002] = {0};

class Solution {
public:
    int minMoves(vector<int>& nums, int limit) {
        const int n = nums.size();
        const int hn = n / 2;
        const int max_x = 2 * limit;

        fill(x_axis, x_axis + max_x + 2, 0);

        for(int i=0; i<hn; ++i){

            int v1 = nums[i];
            int v2 = nums[n-1-i];

            int a = v1 < v2 ? v1 : v2;
            int b = v1 < v2 ? v2 : v1;

            --x_axis[a+1];
            --x_axis[a+b];
            ++x_axis[a+b+1];
            ++x_axis[b+limit+1];
        }

        int reps = n;
        int answer = n;

        for(int x = 2; x <= max_x; ++x){
            reps += x_axis[x];
            if(reps < answer)
                answer = reps;
        }

        return answer;
    }
};
```
