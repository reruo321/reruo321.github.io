The problem requires the return of an `int` vector containing the final health information of robots.

## Idea
Since all robots have the same speed and unique initial position, any pairs of robots would never meet unless they are moving to each other. So the exact positions of robots do not matter, but we just need the relationship between two nearby robots.

## Algorithm
I sorted robots used `stack`.

## Trial
Aceepted / 101 ms (Beats 20.27%) / 216.04 MB (Beats 33.65%)

I created a custom `struct`, `Robot`. And I also created a `Robot` vector and sorted it by positions using a lambda expression. Finally, I used a `stack` to manage alive robots.

```cpp
struct Robot {
    int no;
    int pos;
    int hp;
    char dir;

    Robot(int no, int pos, int hp, char dir): no(no), pos(pos), hp(hp), dir(dir) {}
    Robot(): no(0), pos(0), hp(0), dir('L') {}
};

bool pos_comp(const Robot &a, const Robot &b) {
    return a.pos < b.pos;
}

bool no_comp(const Robot &a, const Robot &b){
    return a.no < b.no;
}

class Solution {
public:
    vector<int> survivedRobotsHealths(vector<int>& positions, vector<int>& healths, string directions) {
        vector<Robot> r_vec;
        const int n = positions.size();

        r_vec.assign(n, Robot());

        for(int i=0; i<n; ++i){
            r_vec[i] = Robot(i, positions[i], healths[i], directions[i]);
        }
        sort(r_vec.begin(), r_vec.end(), pos_comp);

        stack<Robot> r_stack;

        for(int i=0; i<n; ++i){
            if(r_stack.empty()){
                r_stack.push(r_vec[i]);
            }
            else{
                Robot right = r_vec[i];
                bool ispush = false;

                while(1){
                    if(!r_stack.empty()) {
                        // Collide
                        Robot left = r_stack.top();

                        if(left.dir == 'R' && right.dir == 'L'){
                            r_stack.pop();
                            // Left
                            if(left.hp > right.hp){
                                r_stack.push(Robot(left.no, left.pos, left.hp - 1, left.dir));
                                break;
                            }

                            // Right
                            else if(left.hp < right.hp){
                                --right.hp;
                                continue;
                            }

                            // None
                            else {
                                break;
                            }
                        }
                    }
                    r_stack.push(right);
                    break;
                    // end of while
                }
            }
        }
        vector<Robot> final_robot_vec;

        while(!r_stack.empty()){
            Robot t = r_stack.top();
            final_robot_vec.push_back(t);
            r_stack.pop();
        }
        sort(final_robot_vec.begin(), final_robot_vec.end(), no_comp);
        vector<int> answer_vec;

        for(auto &r: final_robot_vec){
            answer_vec.push_back(r.hp);
        }
        return answer_vec;
    }
};
```

## Optimization
### Revision 1
Added an optimization trick for programming tests. In LeetCode, wrap them to a `static const int` lambda expression with IIFE(Immediately Invoked Function Expression). The program will run the optimization immediately and destroy the expression before the main function.

```cpp
static const int fast_io = [](){
    ios::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);
    return 0;
}();   // This is IIFE without any arguments.
```

### Revision 2
Use **indirect sorting**. Do not define `struct Robot` to manage the index, the position, the health, and the direction of a robot together. Instead, just make an `int` vector `indices` that contains robot indices and would be sorted in the order of robots' positions. We need to use another lambda expression whose caputure clause has `&` to capture any variables inside the main function.

![indices](indices.png)

```cpp
int n = directions.size();

auto pos_comp = [&](const int &a, const int &b){   // [&] captures external variables by reference. It captures the positions vector in the main function scope.
    return positions[a] < positions[b];
};

vector<int> indices(n, 0);
iota(indices.begin(), indices.end(), 0);   // iota assigns numbers from 0 to n-1 to the vector.
sort(indices.begin(), indices.end(), pos_comp);
```

### Revision 3
Instead of using `stack`, use `vector`. The `vector` storing its elements in a contiguous block of memory is faster than the `stack` which the chunked memory of `deque`, the default behind the `stack`. The contiguous block of memory can increase cache locality and reduce cache misses.

To frequently push and pop and avoid vector resizing, use `reserve()` to pre-allocate memory.

```cpp
vector<int> idx_vst;
idx_vst.reserve(n);
```

### Revision 4
Instead of checking whether the stack vector `idx_vst` is empty or not first, check if an input robot's direction is 'R'. If it is 'R', just store it to `idx_vst`. Otherwise, check the direction of the latest robot of `idx_vst`. Remove any removed robots by popping it from `idx_vst` or marking the input robot's health to 0. And put any survivor into `idx_vst`.

```cpp
for(int i: indices){
    if(directions[i] == 'R'){
        idx_vst.push_back(i);
    }
    else{
        while(!idx_vst.empty() && directions[idx_vst.back()] == 'R' && healths[i] > 0){
            if(healths[idx_vst.back()] < healths[i]){
                --healths[i];
                idx_vst.pop_back();
            }
            else if(healths[idx_vst.back()] > healths[i]){
                --healths[idx_vst.back()];
                healths[i] = 0;
            }
            else{
                idx_vst.pop_back();
                healths[i] = 0;
            }
        }
        if(healths[i] > 0){
            idx_vst.push_back(i);
        }
    }
}
```

### Revision 5
Instead of sorting the stack vector to get the list of survivors, use **counting sort** that increases the values of specific indices. While the former `sort()` takes $O(K log K)$ where $K$ is the number of survivors, the latter takes $O(N)$ where $N$ is the number of robots.

![answer](answer.png)

1. Create a new vector `hp_vec` whose size is the number of robots and filled with `-1`.
2. Assign the survivors' healths to `hp_vec` elements with corresponding indices.
3. Create the answer vector `answer` whose size is the number of survivors.
4. Copy all positive healths `hp_vec` to `answer` by reading elements from `0`-th to `(n-1)`-th.

```cpp
int k = idx_vst.size();
vector<int> answer;
answer.reserve(k);

for(int i: idx_vst){
    hp_vec[i] = healths[i];
}

for(int h: hp_vec){
    if(h > 0){
        answer.push_back(h);
    }
    if(answer.size() == k)
        break;
}

return answer;
```

class Solution {

    vector<int> survivedRobotsHealths(vector<int>& positions, vector<int>& healths, string directions) {

        auto pos_comp = [&](const int &a, const int &b){
            return positions[a] < positions[b];

        sort(indices.begin(), indices.end(), pos_comp);

        vector<int> idx_vst;
        idx_vst.reserve(n);

        for(int i: indices){
            if(directions[i] == 'R'){
                idx_vst.push_back(i);
            }
            else{
                while(!idx_vst.empty() && directions[idx_vst.back()] == 'R' && healths[i] > 0){
                    if(healths[idx_vst.back()] < healths[i]){
                        --healths[i];
                        idx_vst.pop_back();
                    }
                    else if(healths[idx_vst.back()] > healths[i]){
                        --healths[idx_vst.back()];
                        healths[i] = 0;
                    }
                    else{
                        idx_vst.pop_back();
                        healths[i] = 0;
                    }
                }
                if(healths[i] > 0){
                    idx_vst.push_back(i);
                }
            }
        }
